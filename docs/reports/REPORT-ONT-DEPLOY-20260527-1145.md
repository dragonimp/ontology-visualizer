## 4. 遇到的问题与风险

### 问题 1：`\n` 字面量导致 JS 语法错误
- **严重程度**：🔴 高
- **原因**：`patch` 工具在处理多行代码替换时，将换行符转义为字符串 `\n`
- **修复**：移除字面量 `\n`，恢复为真实换行
- **影响**：导致整个页面 JS 解析失败，卡在加载界面

### 问题 2：CSS `.hidden` 类缺少 `display: none`
- **严重程度**：🔴 高
- **原因**：CSS 只设置了 `opacity: 0; pointer-events: none;`，未清除 `display: flex`
- **修复**：添加 `display: none !important`
- **影响**：遮罩层虽不可见，但仍占据整个屏幕，阻挡 canvas 交互
- **教训**：CSS 隐藏元素时，必须同时设置 `opacity` 和 `display`

### 风险
| 风险 | 严重程度 | 应对方案 |
|------|---------|---------|
| `patch` 工具可能引入 `\n` 转义错误 | 中 | 部署前必须检查 `grep -c '\\n'` |
| 服务器文件被意外破坏 | 高 | 禁止直接编辑，始终下载→本地→上传 |
| 大文件（>5MB）加载较慢 | 中 | 已优化批量渲染参数 |
